## 关键注意点

### 1. 不要整包覆盖 `src-tauri/src/proxy/providers/mod.rs`

上游 `v3.15.0` 在 provider 结构上比旧分支新，之前如果直接把旧文件整包搬过来，会破坏 upstream 新增的 Codex Chat 相关导出。

当前正确做法是：

- 以 upstream 的 `mod.rs` 为准
- 只重新应用默认 endpoint 修改

### 2. 版本号不能回退

旧分支部分文件还停在 `3.14.1`。迁移时已经恢复到上游版本 `3.15.0`，后续再改动时不要把以下文件又带回旧版本号：

- `src-tauri/Cargo.toml`
- `src-tauri/Cargo.lock`
- `src-tauri/tauri.conf.json`

### 3. 本地不要提交私钥

以下文件目前仅本地使用，不应提交：

- `.tauri-updater.key`
- `.tauri-updater.key.pub`

## 本地开发运行结论

### 1. Node 版本要求

本项目当前依赖的 `Vite 7.3.0` 不能使用 `Node 20.5.0`。

已验证可用版本：

- `Node 22.12.0`

### 2. 当前机器上遇到过的问题

- 默认 `pnpm` 可能经由 `corepack` 访问 `registry.npmjs.org`
- 受网络或环境限制时，`corepack` 会导致 `pnpm tauri dev` 启动失败
- 本机最终采用的是 `Node 22.12.0 + /usr/local/bin/pnpm`

### 3. 前端访问方式

本地前端开发服务可通过：

- `http://localhost:3000/`

注意这次服务监听在 `localhost / ::1`，不是 `127.0.0.1`，所以如果访问 `127.0.0.1:3000` 可能会误以为服务已停。
